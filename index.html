Concept Preview - February 2026

AIを、冒険にしよう。

タスクをRPGのクエストのように進める、
非エンジニア向けAIワークフローアプリ

子供からおばあちゃんまで。
誰でも30秒でAIの成果物が手に入る。

Problem

AIは便利。でも、使えない人が多い。

ChatGPTやClaude Codeは高機能。でも非エンジニアにとって、AIツールには見えない壁がある。

😱

心理的恐怖

ターミナル = ハッカーの画面。「何か壊しそう」という恐怖が、最初のステップを阻む。

🛠

環境構築の壁

Node.js? API key? 環境変数? 最初の「Hello World」にたどり着く前に離脱する。

❓

何を入力すべきか不明

プロンプト? コマンド? フリーテキストの空白テキストボックスは、初心者にとって恐怖。

📄

結果の受け取り方

JSONが返ってきても読めない。ファイルがどこに保存されたかもわからない。

Concept

ワークフローを「設計する」のではなく、
クエストを「冒険する」。

AIの専門用語をゲームの言葉に置き換える。それだけで、誰でもAIが使えるようになる。

ゲーム内の体験 実際にやっていること
⚔ クエストを受ける → タスクを選ぶ
🪄 呪文の材料を入れる → フォームに入力する
✨ 魔法が発動する → AIがタスクを実行する
📜 魔法の成果物を受け取る → 結果をダウンロードする
⭐ レベルアップ! → 新しいタスクカテゴリが解放される
Dify/n8n/Zapierは「何でもできるが設定が難しい」。
このアプリは「できることは限られるが、誰でも30秒で完成物が手に入る」。

Character Design

二軸のキャラクターシステム

AIの化身（マスコット）と冒険者の分身（アバター）。性別ではなく役割で選ぶ。
Stardew Valley / Undertale風の16bitクリアピクセルアート。

🐈‍⬛ マスコット（AIの化身）

黒猫 = 知的・戦略系。「分析するにゃ」
ラブラドール = 元気・行動系。「やるぞ!」

クエスト実行中の進捗報告、ヒント、お祝いを担当。

🧙 アバター（ユーザーの分身）

策士（Planner） = 分析・計画系クエスト向き
探求者（Explorer） = リサーチ・発見系クエスト向き
創作者（Crafter） = クリエイティブ系クエスト向き

クエストクリアで成長: 色 → 装備 → シルエット変化

⭐ キャラクター成長チャート

クエストをクリアするたびにキャラクターが成長。3段階の変化で達成感を演出。

Differentiation

既存ツールとの違い

🛠 Dify / n8n / Zapier

  • 👥 エンジニア・テック寄り
  • 🖥 ノード+線のフローチャートUI
  • ⚙ ワークフローを「設計する」
  • 📝 パラメータ設定が必要
  • ⚡ モチベーション = 効率化
  • ♻ 価値 = 自動化の自由度

✨ Quest App (This)

  • 👪 非エンジニア全般（子供〜高齢者）
  • 🎮 ゲーム風カードUI
  • ⚔ クエストを「冒険する」
  • 📋 ガイド付き選択式入力
  • 🌟 モチベーション = 楽しさ + 成長
  • 🎯 価値 = 完成物への最短距離
